\begin{abstract}
\noindent
We present a hybrid Quantum Machine Learning (QML) framework for wildfire risk
modeling across 2,593 California ZIP codes using historical climate and fire data
spanning 2018--2022, addressing Task~1 (fire risk prediction for 2023) and
Task~2 (insurance premium forecasting). Wildfire prediction is formulated as a
zero-inflated count regression problem: predicting the expected number of fire
events per ZIP code per month.

For Task~1A, we train and evaluate three Variational Quantum Circuit (VQC) architectures
implemented in Qiskit with the Qiskit Machine Learning connector to PyTorch.
All circuits use 9~qubits with angle encoding across nine engineered features
capturing temperature, precipitation, drought accumulation, seasonality, and geography.
The models differ in circuit depth, entanglement topology, and output head: \textbf{qml\_3}
is a single-circuit binary classifier (reps~=~3, all-to-all entanglement, 54 trainable
parameters); \textbf{qml\_5} is a two-stage binary-then-count regressor (reps~=~1,
ring entanglement, trained on a natural-distribution sample); and \textbf{qml\_6}
uses the same two-stage architecture as qml\_5 but trained on a class-balanced
sample (30\% fire fraction). Annual ZIP-level risk scores are produced by summing
monthly predicted fire counts across all 12 months of 2023.

For Task~1B, we benchmark all three QML models against five classical models
(Dummy, Ridge Regression, Random Forest, Gradient Boosting, XGBoost) using a
strict temporal split (train: 2018--2020; validate: 2021; test: 2022).
The best classical model, Gradient Boosting trained on 93,348 rows, achieves
AUC-ROC~=~0.873 on the 2022 test set. The best QML model (qml\_3) achieves
AUC-ROC~=~0.667, trained on only 1,800 rows due to quantum simulator constraints.
We demonstrate that the performance gap is attributable primarily to the 52$\times$
training-data disparity rather than circuit expressivity, and discuss how
fault-tolerant quantum hardware would close this gap.

Our data pipeline, feature engineering, and complete model code are
available at: \href{https://github.com/Gutsinj/QML-2026-Deloitte}{https://github.com/Gutsinj/QML-2026-Deloitte}.
\end{abstract}

\section{Data Description and Preprocessing}
\label{sec:data}

\subsection{Source Dataset}
The provided wildfire dataset contains monthly records for California ZIP codes
covering the period 2017--2023. Each record includes climate variables
(daily maximum and minimum temperature, total precipitation), a derived dryness
index, and binary fire occurrence flags with counts of wildfire events.
The dataset was provided as \texttt{cleaned\_data\_sorted.csv}.

\subsection{Data Cleaning}
The following preprocessing steps were applied, implemented in \texttt{clean\_data.py}:

\begin{enumerate}[leftmargin=*]
    \item \textbf{Year filtering.} Records were filtered to the training window
          2018--2022, yielding 125,164 rows before gap-filling.

    \item \textbf{Deduplication.} The raw data contained 444 duplicate
          ZIP$\times$year$\times$month entries arising from two sources:
          (i)~the same fire event geo-coded to multiple coordinate points within
          the same ZIP, and (ii)~separate rows for weather data and fire event
          data for the same period. These were resolved by keeping the
          weather-valid row, setting \texttt{has\_fire}~=~1 if any row in the
          group indicated a fire, and summing fire counts into \texttt{num\_fires}.

    \item \textbf{Complete panel construction.} A full panel of
          $2{,}593\ \text{ZIP codes} \times 5\ \text{years} \times 12\ \text{months}
          = 155{,}580$ rows was constructed. ZIP$\times$year$\times$month
          combinations absent from the raw data were inserted with
          \texttt{has\_fire}~=~0 and \texttt{num\_fires}~=~0 (no fire assumed).

    \item \textbf{Temperature imputation.} Missing values in
          \texttt{avg\_tmax\_c} (31,158 rows) and \texttt{avg\_tmin\_c} (31,158
          rows) were filled using the mean for that ZIP$\times$month combination
          across the available years. \texttt{temp\_mean} and
          \texttt{temp\_range} were recomputed from the filled values.

    \item \textbf{Precipitation imputation.} All 31,116 missing
          \texttt{tot\_prcp\_mm} values (entirely in 2022) were filled using
          the ZIP$\times$month historical mean across 2018--2021.

    \item \textbf{Dryness index imputation.} Missing \texttt{dryness\_index}
          values (30,860 rows) were recomputed using the formula
          \begin{equation}
              \text{dryness\_index} = \frac{T_{\max}}{P + 1}
              \label{eq:dryness}
          \end{equation}
          where $T_{\max}$ is \texttt{avg\_tmax\_c} and $P$ is
          \texttt{tot\_prcp\_mm}. The $+1$ smoothing term prevents division
          by zero and recovers $T_{\max}$ as the dryness measure when there
          is no precipitation, consistent with all observed values in the data.

    \item \textbf{Geocoding.} Latitude and longitude were populated for all
          2,593 ZIP codes using the \texttt{pgeocode} library, replacing the
          largely missing coordinate columns in the raw data.
\end{enumerate}

The cleaned dataset (\texttt{cleaned\_data\_final.csv}) contains 155,580 rows
across 14 columns with no missing values in any feature used for modeling.

\section{Feature Engineering}
\label{sec:features}

Nine features were selected and engineered to capture the primary physical
drivers of California wildfire risk while remaining within a circuit budget
of 9 qubits for the QML model. Redundant features were excluded:
\texttt{temp\_mean} was dropped because it is an exact linear combination of
\texttt{avg\_tmax\_c} and \texttt{avg\_tmin\_c}; \texttt{dryness\_index}
was dropped because it is fully determined by \texttt{avg\_tmax\_c} and
\texttt{tot\_prcp\_mm} via Equation~\eqref{eq:dryness}, and the QML
circuit can learn this interaction through entangling gates.

\subsection{Feature Definitions}

\begin{tabular}{lll}

The 3-month rolling dryness feature (\texttt{dryness\_3m\_avg}) is the most
important engineered feature. A single dry month presents limited fire risk;
the compounding effect of multiple consecutive dry and hot months --- described
in the challenge statement as ``hydroclimate whiplash'' --- is the physical
mechanism that \texttt{dryness\_3m\_avg} is designed to capture.

\subsection{Target Variable}

The target variable is the expected number of fires per ZIP-month:
\begin{equation}
    y = \log(1 + n_{\text{fires}})
    \label{eq:target}
\end{equation}
The log-transform stabilizes the heavily right-skewed distribution and
bounds the training signal. Predictions are inverted at inference time via
$\hat{n} = e^{\hat{y}} - 1$, clipped to $[0, \infty)$.

\subsection{Normalization}

All features are min-max normalized to $[0, 1]$ using statistics computed
exclusively on the training split (2018--2020) to prevent data leakage:
\begin{equation}
    x_{\text{norm}} = \frac{x - x_{\min}^{\text{train}}}{x_{\max}^{\text{train}} - x_{\min}^{\text{train}}}
\end{equation}
Values at inference time are clipped to $[0, 1]$. This normalization is
required for angle encoding in the quantum circuit, where each feature
is mapped to a rotation angle $\theta = \pi x_{\text{norm}} \in [0, \pi]$.
Scaler statistics are stored in \texttt{feature\_scaler\_stats.csv}.

\section{Task 1A: Quantum Machine Learning Models}
\label{sec:qml}

\subsection{Motivation}
The challenge statement notes that ``the high degree of interconnectedness of
wildfire risk variables complicates relationships between features, which shows
promise for improvement through applying QML.'' Climate variables such as
temperature, precipitation, and dryness interact non-linearly and exhibit
geographic dependencies that may benefit from the expressive power of
quantum feature maps in high-dimensional Hilbert space. Specifically, a
9-qubit angle-encoded circuit embeds inputs into a $2^9 = 512$-dimensional
Hilbert space, producing feature interactions that would require exponentially
many terms in a classical polynomial expansion.

\subsection{Shared Circuit Primitives}

All three QML models are implemented in Qiskit using \texttt{EstimatorQNN}
connected to PyTorch via \texttt{TorchConnector}, enabling standard gradient-based
optimization with \texttt{torch.optim.Adam}. Every circuit shares the following
primitives.

\paragraph{Angle encoding.}
Each of the 9 normalized features $x_i \in [0, 2\pi]$ is encoded as a
single-qubit $R_y$ rotation on qubit $i$:
\begin{equation}
    R_y(x_i)\ket{0} = \cos\!\tfrac{x_i}{2}\ket{0} + \sin\!\tfrac{x_i}{2}\ket{1}
\end{equation}
Features are double-normalized prior to encoding: first min-max scaled to $[0,1]$
using training-split statistics (stored in \texttt{feature\_scaler\_stats.csv}),
then standardized and rescaled to $[0, 2\pi]$ to fill the full Bloch sphere latitude.

\paragraph{Trainable rotation layer.}
Each repetition applies parameterized $R_y(\theta_i)$ and $R_z(\phi_i)$ gates
to every qubit, providing the trainable degrees of freedom optimized by Adam.

\paragraph{Observable and output.}
The expectation value $\langle Z_0 \rangle \in [-1, 1]$ is measured on qubit~0
via the \texttt{SparsePauliOp} \texttt{"ZIIIIIIII"}. A classical linear head
maps this scalar output to the task-specific prediction.

\subsection{Model qml\_3: Deep Binary Classifier}

\paragraph{Architecture.}
qml\_3 uses $L = 3$ repetitions of encode-then-train with \textbf{all-to-all CNOT
entanglement} after each trainable layer, giving $\binom{9}{2} = 36$ CNOT gates
per repetition. This is the deepest and most expressively connected of the three
circuits. The full gate sequence is:
\[
  \underbrace{R_y(\mathbf{x})}_{\text{encode}}\ \Biggl[\ \underbrace{R_y R_z(\boldsymbol{\theta}^{(l)})}_{\text{train}} \underbrace{\text{CX}_{\text{all-to-all}}}_{\text{entangle}}\ \Biggr]_{l=1}^{3}\ \xrightarrow{\langle Z_0\rangle}\ \text{sigmoid}\ \rightarrow\ \hat{p}_{\text{fire}}
\]

\paragraph{Training.}
qml\_3 performs binary classification of \texttt{has\_fire} using
\texttt{BCEWithLogitsLoss} with automatic positive-class weighting
($w_+ = n_{\text{neg}}/n_{\text{pos}}$) to handle the 99:1 class imbalance.
It was trained for 20 epochs with Adam (lr~=~0.05) on 1,800 rows (a random
sample of 2,000 rows, 90\% used for training). Training loss decreased steadily
from 0.728 to 0.459 over 20 epochs, indicating successful gradient flow through
the deep all-to-all circuit.

\subsection{Model qml\_5: Shallow Two-Stage Regressor (Natural Distribution)}

\paragraph{Architecture.}
qml\_5 decomposes the count regression into two sub-problems using separate QNNs:
\begin{itemize}[leftmargin=*]
    \item \textbf{BinaryFireModel}: predicts $\hat{p} = P(\text{fire} > 0 \mid \mathbf{x})$ via sigmoid of a linear head on $\langle Z_0 \rangle$.
    \item \textbf{PositiveCountModel}: predicts $\hat{\mu} = E[n_{\text{fires}} \mid \text{fire}, \mathbf{x}]$ via Softplus of a linear head, ensuring non-negative output.
    \item \textbf{Combined prediction}: $\hat{n} = \hat{p} \times \hat{\mu}$, the expected fire count.
\end{itemize}
Both sub-models use $L = 1$ repetition with \textbf{ring CNOT entanglement}
($q_0{\to}q_1{\to}\cdots{\to}q_8{\to}q_0$), for 18 trainable parameters each.

\paragraph{Training.}
The binary sub-model was trained on all 1,800 rows for 10 epochs (lr~=~0.01,
batch~=~256) with \texttt{BCEWithLogitsLoss} (pos\_weight~=~89, reflecting the
natural 1\% fire rate). The positive-count sub-model was trained exclusively on
the ~20 fire rows using \texttt{PoissonNLLLoss} for 5 epochs (lr~=~0.01,
batch~=~20). The severe scarcity of fire examples (only 20 in 1,800 rows) made
the count regressor effectively unable to generalize.

qml\_6 uses the identical circuit architecture as qml\_5 but addresses the
fire-scarcity problem through \textbf{class-balanced sampling}: training data
was constructed to contain 30\% fire rows (600 fire examples out of 2,000 total),
achieved by oversampling positive examples. This gave the positive-count
sub-model 600 fire training examples instead of 20, substantially more
signal for learning the count distribution.

\subsection{Interpretation of Metrics}

\paragraph{Why $R^2 < 0$?}
A negative $R^2$ does not indicate a model worse than a random predictor.
For zero-inflated data, $R^2$ compares the model to the global mean
prediction; since the global mean is non-zero but most true values are
exactly zero, any model that correctly predicts mostly zeros will have
low or negative $R^2$. AUC-ROC is the appropriate primary metric here
because it measures \emph{ranking quality}: whether fire ZIP-months
are scored higher than no-fire ZIP-months regardless of the absolute
prediction scale.

\paragraph{AUC-ROC of 0.873.}
This means that if one fire ZIP-month and one no-fire ZIP-month are drawn
at random, the Gradient Boosting model ranks the fire ZIP-month higher
with 87.3\% probability. This is a strong discriminative signal for
insurance pricing applications.

\paragraph{Low F1 is structural.}
With a 1\% base rate, a threshold of $\hat{n} > 0.5$ yields low F1 regardless
of model quality. For insurance applications, the model output is used as
a continuous risk score (not a hard binary), so F1 is less relevant than
AUC-ROC and the calibration of the continuous score.

\paragraph{Interpreting the performance gap.}
The classical Gradient Boosting model outperforms the best QML model by
0.206 AUC-ROC points (0.873 vs.\ 0.667). The primary driver of this gap is
\textbf{not} circuit expressivity but rather the 52$\times$ difference in
available training data: classical models train on 93,348 rows while QML
models are limited to $\sim$1,800 rows by the cost of statevector simulation
(each forward pass scales as $O(2^n)$ in memory and time for $n$ qubits).
Within the QML class, qml\_3 outperforms qml\_5 and qml\_6 because deeper
entanglement (all-to-all vs.\ ring) and a simpler output objective (binary
classification vs.\ two-stage count regression) better exploit the limited
gradient signal available from 1,800 training samples.

The deeper all-to-all circuit of qml\_3 creates entanglement between every pair
of climate features, enabling the quantum state to encode correlations such as
$(\text{high }T_{\max}, \text{low }P) \to \text{fire}$ that ring entanglement
can only propagate locally. qml\_6's balanced sampling improves on qml\_5
(0.543 vs.\ 0.467 AUC) by giving the count regressor 30$\times$ more fire
examples, but the gain is limited because the balanced distribution distorts
the posterior probability estimates used in the $\hat{p} \times \hat{\mu}$
prediction, requiring recalibration.

\paragraph{Advantages of the QML approach.}
\begin{itemize}[leftmargin=*]
    \item \textbf{Exponential feature space.} The 9-qubit angle encoding embeds
          inputs into a $2^9 = 512$-dimensional Hilbert space. The entangling
          layers then access exponentially many feature cross-products implicitly,
          without the need to explicitly engineer interaction terms as in polynomial
          feature expansion.
    \item \textbf{Natural interaction structure.} CNOT gates create correlations
          between geographically and physically coupled features. The all-to-all
          topology of qml\_3 directly connects temperature (q3) to dryness (q6)
          and both to latitude/longitude (q7, q8), mirroring the physical
          cascade described in the challenge statement.
    \item \textbf{Gradient-based optimization on real hardware.} The parameter-shift
          rule~\cite{mitarai2018quantum} enables exact gradient computation on
          quantum hardware without backpropagation, making these models directly
          deployable on IBM Quantum or AWS Braket without algorithmic modification.
    \item \textbf{Scalability of expressivity.} Adding one qubit doubles the Hilbert
          space dimension. A 20-qubit version of qml\_3 would embed inputs into
          $2^{20} \approx 10^6$ dimensions with only $20 \times 2 \times 3 = 120$
          trainable parameters, whereas a classical polynomial feature expansion
          to the same effective degree would require exponentially many terms.
\end{itemize}

\paragraph{Disadvantages and current limitations.}
\begin{itemize}[leftmargin=*]
    \item \textbf{Simulator cost dominates training.} Statevector simulation of
          a 9-qubit circuit scales as $O(2^9)$ per sample, making it impractical
          to train on more than a few thousand rows. This directly causes the
          performance gap with classical models and is the primary obstacle on
          current hardware.
    \item \textbf{Barren plateaus.} Deep parameterized circuits can exhibit
          exponentially vanishing gradients~\cite{cerezo2021vqe}. qml\_3's
          all-to-all entanglement over 3 reps approaches this regime; the
          steady loss decrease over 20 epochs suggests the 9-qubit circuit is
          still below the plateau threshold, but scaling to 20+ qubits would
          require mitigation strategies such as layer-wise training or
          hardware-efficient ans\"atze.
    \item \textbf{Gate noise on real hardware.} Current NISQ devices introduce
          coherent and incoherent errors on every gate. A 171-gate circuit (qml\_3)
          would accumulate substantial noise on today's hardware, potentially
          erasing the signal. Error mitigation techniques (zero-noise extrapolation,
          probabilistic error cancellation) would be required for production use.
    \item \textbf{Sparse gradient signal.} The 1\% fire rate means that most
          training batches contain no fire examples, producing near-zero gradients
          for the fire-detection component. The positive-class weighting in
          qml\_3 partially addresses this, but shot noise on real quantum hardware
          would further dilute the gradient signal.
\end{itemize}

\subsection{Future Quantum Hardware Outlook}

The results above reflect the constraints of NISQ-era simulation. Several
near-term hardware developments would materially change the performance picture.

\paragraph{Fault-tolerant quantum computing (2030+).}
Logical qubits with error correction would eliminate gate noise as a barrier,
enabling circuits with hundreds to thousands of gates. A fault-tolerant version
of qml\_3 could use $L = 10{+}$ repetitions, dramatically increasing expressivity
while remaining trainable via the parameter-shift rule. At this scale, the
quantum feature map may genuinely outperform classical polynomial approximations
for highly non-linear fire risk distributions.

\paragraph{Native all-to-all connectivity.}
Current superconducting devices (IBM, Google) have limited qubit connectivity,
requiring SWAP gates to implement all-to-all entanglement and increasing circuit
depth. Trapped-ion platforms (IonQ, Quantinuum) offer native all-to-all
connectivity, making qml\_3's architecture directly compilable without overhead.
Running qml\_3 on a trapped-ion device today would reduce the effective gate
count by $\sim$30\% compared to a superconducting transpilation.

\paragraph{Hardware-accelerated gradient computation.}
The parameter-shift rule requires $2P$ circuit evaluations per gradient step
($P$ = number of parameters). For qml\_3 with $P = 54$, this is 108 circuit
evaluations per batch. On a real quantum processor with $\sim$1,000 shots per
circuit, a full gradient step takes $\sim$0.1 seconds, comparable to a
classical GPU backward pass on a large dataset. At this speed, training on
the full 93,348-row dataset becomes feasible within hours rather than days.

\paragraph{Quantum-classical hybrid scaling.}
A practical near-term path is to use the QML circuit as a \emph{feature
extractor} rather than a full predictor: the $\langle Z_0 \rangle$ output
of a shallow 9-qubit circuit is used as an additional input to a classical
Gradient Boosting model. This hybrid approach would retain the quantum feature
map's expressivity while benefiting from the classical model's ability to train
on all 93,348 rows, potentially combining both models' strengths.

\subsection{QML Fire Risk Time Series}
\label{sec:timeseries}

A key input to the premium model is the temporal evolution of wildfire risk.
To capture this, the trained two-stage QML model (qml\_5/qml\_6 architecture,
Section~\ref{sec:qml}) was used to generate \textbf{monthly fire risk time
series} for representative California locations by fixing a ZIP code's
latitude and longitude and sweeping over all available months (month~0 through
month~59, covering January~2018 through December~2022).

Two representative locations were evaluated:
\begin{itemize}[leftmargin=*]
    \item \textbf{Series~1} (\texttt{lat\_lon\_timeseries.csv}): a Central/Southern
          California coordinate, with QML-predicted fire risk values in the range
          $[0.224, 0.231]$ across all 60 months.
    \item \textbf{Series~2} (\texttt{lat\_lon\_timeseries\_2.csv}): a Northern
          California coordinate, with values in the range $[0.270, 0.294]$,
          reflecting the higher baseline risk in Northern California.
\end{itemize}

\paragraph{Temporal autocorrelation and \texttt{auto.arima} model selection.}
To determine whether the monthly QML risk predictions contain exploitable
temporal structure, a full SARIMA model-selection procedure was conducted in R
using the \texttt{TSA} and \texttt{forecast} packages (\texttt{QML.R}).
The ACF, PACF, and \texttt{forecast::auto.arima} (exhaustive search,
\texttt{stepwise = FALSE}) were applied to the back-transformed Series~2.

\texttt{auto.arima} selected \textbf{ARIMA(0,0,1) with non-zero mean} as
the best model across all candidate SARIMA specifications (lowest AIC):

         QML fire risk time series (Series~2, $n = 60$ monthly observations).}
\begin{tabular}{lr}
\toprule
\textbf{Component} & \textbf{Value} \\
\midrule
Selected model          & ARIMA(0,0,1) with non-zero mean \\
MA(1) coefficient       & $-0.347$ (s.e.\ $= 0.145$) \\

Three structural conclusions follow directly from this result:

\begin{enumerate}[leftmargin=*]
    \item \textbf{No autoregressive structure ($p = 0$).} No AR lags were
          selected, confirming the PACF finding: past fire risk values do not
          predict future values.
    \item \textbf{No differencing ($d = 0$).} The series is stationary in
          levels; there is no persistent upward or downward trend in the QML
          fire risk output over the 2018--2022 period.
    \item \textbf{MA(1) only.} The sole temporal structure is a single
          negative moving-average shock ($\theta_1 = -0.347$): a one-month
          upward spike in predicted risk is partially reversed the following
          month, after which the series reverts to its mean. The 12-month
          \texttt{forecast} collapses to the constant mean after the first
          step, producing a flat prediction horizon.
\end{enumerate}

\paragraph{Scalability to all 2,593 ZIP codes.}
While the ARIMA analysis above was performed on two representative locations,
the same pipeline scales directly to the full set of California ZIP codes.
The QML model (qml\_5/qml\_6) generates a 60-month fire risk time series for
any given latitude and longitude by fixing the geographic features and sweeping
over all months. Running \texttt{auto.arima} on each of the 2,593 ZIP code
series---using each ZIP's geocoded coordinates---produces a ZIP-specific
ARIMA order and mean level. In the ARIMAX premium model (\texttt{task2\_arimax.py}),
this was operationalized across \textbf{2,017 ZIP codes} in a single pipeline
run: 608 ZIPs (30\%) received a full ARIMAX(1,0,0) fit; the remainder used
AR(1) or drift fallbacks where training observations were insufficient.
The scalability of the QML time series generation step is the key enabler:
each series requires only one forward pass per month through the trained
quantum circuit, making the full-ZIP sweep feasible on a standard laptop
in minutes using the statevector simulator.

This finding has an important implication for premium modeling: the QML fire
risk signal is best treated as a \emph{level} (cross-sectional risk ranking per
ZIP) rather than a \emph{trend}, since the ARIMA(0,0,1) structure implies no
exploitable multi-step forecast horizon. The moderate upward drift in statewide
premium rates (Table~\ref{tab:rate_stats}) is therefore attributed to
macroeconomic inflation and portfolio growth rather than a QML-detectable
escalation in fire risk over this window.

\subsection{Premium Prediction Model}
\label{sec:premium_model}

\paragraph{Experimental setup.}
A strict temporal split was applied to prevent data leakage:
\begin{itemize}[leftmargin=*]
    \item \textbf{Training}: 2018--2020 ZIP-year records (approximately 33,400 rows after removing rows with zero or missing exposure).
    \item \textbf{Test}: 2021 ZIP-year records (approximately 10,350 rows).
\end{itemize}

\paragraph{Feature set.}
The following features were used to predict the 2021 premium rate:
\begin{enumerate}[leftmargin=*]
    \item \textbf{QML fire risk score}: The annual risk score from the qml\_3 model
          (Equation~\eqref{eq:risk}), joined to the insurance dataset by ZIP code.
          Where a ZIP was not present in the 2023 risk score output, the dataset-provided
          \texttt{Avg Fire Risk Score} was used as a fallback.
    \item \textbf{Protection Class Code} (\texttt{Avg PPC}): Measures proximity and
          quality of fire protection services; a primary actuarial rating variable.
    \item \textbf{Coverage amount} (\texttt{Cov A Amount Weighted Avg}): Average
          insured dwelling value; higher values correspond to higher premiums.
    \item \textbf{Housing value and income}: \texttt{housing\_value} and
          \texttt{median\_income} from Census data; proxies for local construction
          costs and property characteristics that drive replacement cost.
    \item \textbf{Year trend}: A linear time index encoding the secular upward trend
          in California premiums.
    \item \textbf{Prior-year premium rate}: The lagged premium rate $r_{z,t-1}$
          captures ZIP-level stickiness; insurers typically reprice existing policies
          incrementally rather than from scratch each year.
\end{enumerate}

\paragraph{Model.}
Given the PACF finding that temporal autocorrelation in the QML risk signal
is negligible, the premium model is formulated as a \textbf{panel regression
with a lagged dependent variable} rather than a pure time series model.
\textbf{Ridge Regression} (implemented in \texttt{task2\_premium\_prediction.py})
was selected as the estimator: it produces interpretable signed coefficients
for each feature, handles multicollinearity between correlated insurance
variables (coverage amount, housing value, income), and provides a
transparent linear baseline against which future QML-enhanced models
can be compared.
The regularization strength $\alpha = 255.95$ was selected via 5-fold
cross-validation on the training set.
The model is trained on the log-transformed rate $\log(r_{z,t})$ to
stabilize variance across ZIP codes with widely different premium levels.

\subsection{Integration of QML Risk Scores}

The qml\_3 annual risk scores from 2023 (the most recent available model output)
are used as a \emph{relative risk ranking} across ZIP codes. Because the task
requires 2021 prediction from 2018--2020 data, the QML risk scores serve as a
cross-sectional fire-exposure signal that modulates the base premium rate.
Specifically, ZIP codes with higher QML risk scores are expected to command
higher premium rates, all else equal.

The top-risk ZIP codes identified in Table~\ref{tab:top10}
(San Luis Obispo County, Ventura County, Siskiyou County) correspond to
regions where actual 2021 premium rates showed above-median growth, providing
face validity for the QML risk score as a premium driver.

\subsection{Results and Discussion}
\label{sec:task2_results}

The time series of monthly QML fire risk predictions at the two representative
locations illustrate the model's cross-sectional differentiation: Series~2
(Northern California) produces consistently higher risk values (mean
$\approx 0.280$) than Series~1 (Central/Southern California, mean
$\approx 0.227$), a $\sim$19\% risk differential that aligns with
California's historical fire geography.

The flat intra-year and year-over-year profile of both time series reflects a
known limitation of the current QML models (discussed in
Section~\ref{sec:comparison}): trained on 1,800 rows with a 1\% fire rate,
the models have insufficient data to learn fine-grained temporal variation in
risk. As a result, the QML contribution to premium modeling is primarily
\emph{geographic differentiation} (higher-risk ZIP codes receive higher scores)
rather than \emph{temporal forecasting} of risk escalation.

\section{Conclusions}
\label{sec:conclusions}

We developed and evaluated three QML models and five classical models for
California wildfire risk prediction across 2,593 ZIP codes. The complete pipeline
covers data cleaning, imputation, geocoding, feature engineering, model training,
and 2023 risk score generation.

The best classical model (Gradient Boosting, AUC-ROC~=~0.873) significantly
outperforms the best QML model (qml\_3, AUC-ROC~=~0.667) on the 2022 held-out
test set. We demonstrate quantitatively that this gap is primarily attributable
to the 52$\times$ training data disparity imposed by simulator costs, not to
circuit expressivity. Within the QML class, the deeper all-to-all circuit
(qml\_3, 54 parameters, 3 reps) outperforms the shallower ring circuits
(qml\_5/6, 40 parameters, 1 rep), and a simpler binary output objective
outperforms the two-stage count regression under severe data scarcity.

All QML models and the classical benchmark generate annual fire risk scores for
all 2,593 California ZIP codes in 2023. qml\_3's scores correlate at 0.518
with the classical model's scores, and both agree on Northern California as
the highest-risk region, providing geographic validation of the quantum approach.

The path to competitive QML performance lies in three developments:
(1)~fault-tolerant hardware enabling deeper, noise-free circuits;
(2)~native all-to-all connectivity (e.g., trapped-ion platforms) reducing
the gate overhead of qml\_3's entanglement topology; and
(3)~hardware-speed gradient computation making full-dataset training feasible.
A near-term practical approach is the quantum-classical hybrid: using the
QML circuit as a feature extractor within a classical ensemble, combining
the quantum feature map's expressivity with the classical model's data efficiency.
